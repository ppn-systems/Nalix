# Poly1305 — High-Performance MAC for .NET (RFC 8439)

**Poly1305** is a cryptographically secure message authentication code (MAC) by Daniel J. Bernstein, used in AEAD protocols (ChaCha20-Poly1305, Salsa20-Poly1305), secure messaging, WireGuard, and TLS 1.3.  
This library provides a zero-allocation, stack-only, incrementally updatable `ref struct` implementation — suitable for AEAD, large data streams, and high-security checks.

- **Namespace:** `Nalix.Shared.Security.Hashing`
- **Class:** `Poly1305` (ref struct)
- **Spec:** [RFC 8439, Section 2.5](https://tools.ietf.org/html/rfc8439#section-2.5)

---

## Features

- 16-byte (128-bit) constant-time tags (no timing attacks)
- Key: 32 bytes (split into `r` & `s`, with RFC clamping)
- Span/native buffer, no heap/GC allocations (can be used for long-lived hashing in hot paths)
- Static one-shot compute and verify APIs — or incremental update/finalize API for streaming
- Security: `.Clear()` erases keys/tags; all internal state is stack-resident

---

## API Overview

| Method/Property                                        | Description                                      |
|--------------------------------------------------------|--------------------------------------------------|
| `Poly1305.KeySize` (`32`)                              | Key length                                       |
| `Poly1305.TagSize` (`16`)                              | Tag output length                                |
| `Poly1305.Compute(key, message)`                       | One-shot MAC, returns 16B tag (buffered)         |
| `Poly1305.Verify(key, message, tag)`                   | True if tag matches (constant-time check)        |
| `new Poly1305(key)`                                    | Create incremental Poly1305 instance (ref struct)|
| `.Update(data)`                                        | Feed more bytes                                  |
| `.FinalizeTag(out tag)`                                | Produce tag, can only call once per instance     |
| `.Clear()`                                             | Erase all key/accumulator state (call when done!)|

All static APIs check argument sizes and run in constant-time (where possible).

## Typical Usage

### One-shot MAC

```csharp
using Nalix.Shared.Security.Hashing;

byte[] key = ...;             // 32 bytes
byte[] msg = ...;             // Data

// Compute tag
byte[] tag = Poly1305.Compute(key, msg);

// Verify tag
bool ok = Poly1305.Verify(key, msg, tag); // constant-time
```

---

### Incremental API (e.g., for large streaming data)

```csharp
Poly1305 mac = new(key); 
try 
{
    mac.Update(headerChunk);
    mac.Update(bodyChunk);
    Span<byte> tag = stackalloc byte[Poly1305.TagSize];
    mac.FinalizeTag(tag);
    // tag is the MAC for (headerChunk||bodyChunk)
}
finally
{
    mac.Clear(); // Always zero keys when done!
}
```

## Security Notes

- **32-byte key:** Use only *fresh* random values for each message/session (NEVER re-use key + nonce combination if using for AEAD)
- Clamping/bitmasking of `r` per RFC 8439 (handled for you)
- Always clear MAC state after use for confidential data
- Never expose partially finalized or incremental internal state
- Tags are *not* encryption: always use with encryption for privacy; Poly1305 only guarantees authenticity/integrity

---

## Error Handling

- Throws `ArgumentException` if keys/tags have incorrect size
- Throws `ObjectDisposedException` if used after `.Clear()`
- Throws if buffers are too small, or instance is misused

## Parameter Reference

| Param      | Size   | Description                       |
|------------|--------|-----------------------------------|
| `key`      | 32 B   | Key (fresh random, RFC clamped)   |
| `message`  | any    | Message/data to MAC               |
| `tag`      | 16 B   | Output tag (should be secret)     |

---

## Internal Details

- Purely stack-only, efficient InlineArray buffer arithmetic
- Implements [2^130 - 5] prime field math; clamping per spec
- Safe for use in cryptographic protocols that require streaming update/finalize model (not just monolithic messages)

---

## Example: AEAD (with ChaCha20/Salsa20)

- Derive Poly1305 key = first 32 bytes of cipher keystream
- MAC transcript = (AAD || pad16 || ciphertext || pad16 || lengths)
- Tag comparison: *constant-time*
