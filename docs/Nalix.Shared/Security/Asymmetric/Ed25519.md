# Ed25519 — Secure, Fast EdDSA Digital Signatures for .NET

**Ed25519** provides efficient, easy-to-use API for public key digital signatures and verification based on the EdDSA algorithm and Curve25519.  
Suited for secure messaging, authentication, E2E crypto, blockchain, zero-trust, and embedded protocols.

- **Namespace:** `Nalix.Shared.Security.Asymmetric`
- **Class:** `Ed25519` (static)
- **Key features:**
  - Fast, RFC-compatible Ed25519 signature and verify
  - Secure against side channel attacks (where possible)
  - Deterministic (no random/nonce needed per signature)
  - Minimal heap allocation for high-throughput usage

---

## API Summary

| Method                                                              | Returns      | Purpose                                        |
|---------------------------------------------------------------------|--------------|------------------------------------------------|
| `Sign(byte[] message, byte[] privateKey)`                           | `byte[]`     | Signs message, returns 64B Ed25519 signature   |
| `Verify(byte[] signature, byte[] message, byte[] publicKey)`        | `bool`       | Returns true if signature is valid             |

- **Private key:** 32 bytes (raw scalar)
- **Public key:**  32 bytes (compressed point)
- **Signature:**   64 bytes (32 bytes R; 32 bytes s scalar)

---

## Typical Usage

### Sign a Message

```csharp
using Nalix.Shared.Security.Asymmetric;

byte[] priv = ...;            // Must be 32 bytes!
byte[] message = ...;         // Any byte array

byte[] signature = Ed25519.Sign(message, priv);
// Send message + signature to recipient
```

### Verify a Signature

```csharp
using Nalix.Shared.Security.Asymmetric;

byte[] signature = ...;  // must be 64 bytes
byte[] message = ...;
byte[] pub = ...;        // must be 32 bytes

bool ok = Ed25519.Verify(signature, message, pub);
// ok == true if the signature is valid
```

---

## Algorithm Details

- Deterministic nonce generation (RFC 8032) — signature never reveals private key if keys are not reused.
- Internal hash: Keccak256 (can be adapted for compatibility with other Ed25519 systems that use SHA-512).
- Signature output: `[R (32 bytes) || s (32 bytes)]`
- Constant-time core math (BigInteger-based, avoid branching where feasible)

---

## Error Handling

- Throws `ArgumentException` if any argument is null, signature is not 64 bytes, or keys are not 32 bytes.
- Throws for malformed input or failure to encode/decode curve points.
- Never returns `null` values.

---

## Security Notes

- **Always protect your private key!** Never log, network, or persist without encryption.
- Never sign the same message with two different keys for security protocols requiring anonymity.
- Before using for interoperable protocols, verify hash and point encoding standards (this code uses Keccak256; tweak if needed).
- The implementation uses side-channel resistant math as far as available in .NET `BigInteger`; consider using hardware or managed C bindings for maximized resistance if you need formal proofs.

## Example: Full Signing & Verification

```csharp
byte[] priv = new byte[32]; // Fill with your private scalar
byte[] pub = ...;           // Public key (32 bytes)
byte[] msg = System.Text.Encoding.UTF8.GetBytes("Hello, world!");

// Sign
byte[] sig = Ed25519.Sign(msg, priv);

// Verify
bool isValid = Ed25519.Verify(sig, msg, pub);
```
